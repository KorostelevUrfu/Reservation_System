# Учебный проект для создания простого REST API сервиса по бронированию
**На данный момент реализовано:** 
* CRUD операции
* Хранение информации в БД (PostgreSQL)
* Возникновение конфликта наложения дат при подтверждении одной из резервации
* Корректная обработка ошибок


## API Endpoints

| Метод | Endpoint | Описание |
|-------|----------|----------|
| `GET` | `/reservation` | Получить список всех резерваций |
| `GET` | `/reservation/{id}` | Получить резервацию по ID |
| `POST` | `/reservation` | Создать новую резервацию |
| `POST` | `/reservation/{id}/approve` | Подтвердить резервацию |
| `PUT` | `/reservation/{id}` | Обновить существующую резервацию |
| `DELETE` | `/reservation/{id}` | Удалить резервацию |
